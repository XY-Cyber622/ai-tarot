## 设计基调

**关键词**：Mystic · Elegant · Dark · Golden · Healing

**核心隐喻**：夜晚的星空 + 古老塔罗 + 烛光 + 流动的星轨

**整体气质**：像走进一家深夜的占卜店 —— 安静、神秘、温暖、有光

---

## 1. Design Principle（设计原则）

### 1.1 七大原则

| 原则 | 描述 | 落地表现 |
| --- | --- | --- |
| **神秘感 Mystic** | 每一个界面都要有"仪式感进入" | 深色背景 + 星点 + 微微发光 + 慢节奏 |
| **优雅 Elegant** | 不堆砌、不噪杂、克制 | 大量留白 + 单一焦点 + 精致细节 |
| **治愈 Healing** | 文案与视觉都要"温柔" | 圆润弧度 + 暖光 + 不刺眼 |
| **沉浸 Immersive** | 用户进入就"出不来" | 全屏背景 + 无干扰元素 + 仪式动效 |
| **金色 Golden** | 金色作为"光的语言" | 关键 CTA、强调、边框、星光 |
| **克制 Restrained** | 不滥用特效 | 单一动效原则、单一焦点原则 |
| **统一 Consistent** | 所有页面同一套语言 | Design Token 化、组件复用 |

### 1.2 设计的"不做什么"

- ❌ 不使用花哨的渐变
- ❌ 不使用大面积彩色
- ❌ 不使用廉价质感的 emoji
- ❌ 不使用强对比刺眼的红绿
- ❌ 不使用动效 > 1.2s（除非仪式性强制流程）
- ❌ 不使用阴影 > 0.5 不透明度

### 4.1 间距 Token

| Token | 数值 | 用途 |
| --- | --- | --- |
| `space-1` | 4px | 最小间距（图标与文字）|
| `space-2` | 8px | 紧凑间距 |
| `space-3` | 12px | 元素内边距（小）|
| `space-4` | 16px | 元素内边距（中）|
| `space-5` | 24px | 元素内边距（大）|
| `space-6` | 32px | 组件间距 |
| `space-7` | 48px | 章节间距 |
| `space-8` | 64px | 页面边距 |
| `space-9` | 96px | 大区块间距 |
| `space-10` | 128px | 仪式性大留白 |

### 4.2 间距使用原则

- **少即是多**：宁可留白，不可堆砌
- **8 的倍数**：核心间距基于 8px 网格
- **呼吸感**：相邻元素至少 16px
- **焦点引导**：用更大间距强调焦点
- **移动端减半**：移动端间距在桌面基础上 -25%

### 4.3 容器尺寸

| 容器中等宽度 | 960px | 100% | 双列内容 |
| 容器最大宽度 | 1200px | 100% | 三列内容 |
| 页面边距 | 64px | 24px | 左右安全距离 |
| 牌堆最大宽度 | 400px | 280px | Draw 页牌堆 |

## 6. Tarot Card Design（塔罗牌设计）

### 6.1 牌的视觉规范

| 属性 | 数值 |
| --- | --- |
| **比例** | 2:3（宽:高），竖版 |
| **桌面尺寸** | 240 × 360 px |
| **移动尺寸** | 160 × 240 px |
| **缩略尺寸** | 80 × 120 px（History） |
| **圆角** | 12px |
| **边框** | 1.5px 金色（带内描边效果）|
| **阴影** | 0 8px 32px rgba(0, 0, 0, 0.5) + 金色光晕 |

### 6.2 牌面结构

```
┌─────────────────┐
│  ┌───────────┐  │  ← 外层：金色边框
│  │  ◊        │  │  ← 牌名（顶部小字）
│  │           │  │
│  │           │  │  ← 主图（中央）
│  │  [牌面]   │  │
│  │           │  │
│  │           │  │
│  │      编号 │  │  ← 编号（底部）
│  └───────────┘  │
│                 │  ← 边距
└─────────────────┘
```

### 6.3 牌面层级

| 层级 | 元素 | 样式 |
| --- | --- | --- |
| L1 | 边框 | 1.5px 金色描边 + 8px 内边距 |
| L2 | 牌面底色 | 浅紫到深紫径向渐变 |
| L3 | 牌名 | 顶部居中，金色，小字号 |
| L4 | 主图 | 中央，占牌面 70% |
| L5 | 编号 | 底部居中，金色装饰字 |

### 6.4 牌背设计

| 元素 | 规范 |
| --- | --- |
| 底色 | 深紫渐变（`#1A1530` → `#0E0B1F`）|
| 图案 | 中心曼陀罗/星芒（SVG，金色描边）|
| 边框 | 同正面 |
| 装饰 | 四角小星点 |

### 6.5 逆位视觉

- 整张牌**整体旋转 180°**
- 顶部出现**小型「↺」标记**（右上角，金色小图标）
- 保持原有阴影 / 边框 / 光晕

### 6.6 牌的悬浮态

| 状态 | 效果 |
| --- | --- |
| 默认 | 阴影 + 微弱金色光晕 |
| Hover | 上浮 8px + 光晕增强 + 缩放 1.03 |
| 选中 | 持续发光 + 边框变粗 2px |
| 翻牌中 | Y 轴 3D 旋转 600ms |
| 锁定 | 透明度 0.6 + 无交互 |

### 6.7 牌组堆叠

- **层叠视觉**：3~5 张牌轻微偏移（每张偏移 2~4px）
- **整体浮动**：牌堆整体有缓慢的悬浮动画
- **可点击反馈**：整叠可点击，点击后顶部牌"飞出"

## 7. Background（背景设计）

### 7.1 背景层次

```
Layer 4（最上）：金色星光粒子（少量，缓慢漂浮）
Layer 3：       紫色光晕（径向渐变，2~3 个，低不透明度）
Layer 2：       流动星轨（曲线，低不透明度，缓慢运动）
Layer 1：       主背景色（深紫黑径向渐变）
Layer 0（最下）：纯色 cosmic black
```

### 7.2 背景 Token

| Token | 描述 | 用途 |
| --- | --- | --- |
| `bg-base` | `#070512` 纯色 | 兜底背景 |
| `bg-radial` | 中心 `#1A1530` → 边缘 `#070512` | 默认页面背景 |
| `bg-card` | `#1A1530` 实色 | 卡片背景 |
| `bg-overlay` | `rgba(7, 5, 18, 0.85)` | 弹窗蒙层 |
| `glow-violet` | 紫色径向光晕，blur 80px | 装饰光晕 |
| `glow-gold` | 金色径向光晕，blur 60px | 焦点光晕 |

### 7.3 背景元素

#### 7.3.1 星空粒子
- **数量**：50~80 个（视性能）
- **颜色**：金色 / 白色 / 淡紫（混合）
- **大小**：1~3px
- **运动**：缓慢漂浮（无终点循环）
- **闪烁**：随机 0.3~1 透明度变化
- **分布**：全屏但底部稀疏、顶部密集

#### 7.3.2 光晕
- **数量**：2~3 个
- **位置**：随机固定位置
- **颜色**：紫色 / 金色（各一）
- **运动**：极慢缩放（5~10s 一次）
- **不透明度**：0.15~0.3

#### 7.3.3 星轨（可选 v1.1）
- **形态**：细弧线
- **数量**：3~5 条
- **颜色**：金色，低不透明度
- **运动**：缓慢沿弧线飘动

### 7.4 移动端背景优化

- 粒子数量减半（性能考虑）
- 减少光晕数量
- 整体不透明度降低 20%

## 8. Animation（动效规范）

### 8.1 缓动函数

| Token | 缓动值 | 用途 |
| --- | --- | --- |
| `ease-mystic` | `cubic-bezier(0.16, 1, 0.3, 1)` | 入场、聚焦（easeOutExpo）|
| `ease-soft` | `cubic-bezier(0.4, 0, 0.2, 1)` | 通用过渡 |
| `ease-bounce` | `cubic-bezier(0.34, 1.56, 0.64, 1)` | 弹性反馈（仅限小元素）|
| `ease-meditate` | `cubic-bezier(0.7, 0, 0.3, 1)` | 长动效（洗牌/解读）|

### 8.2 时长体系

| Token | 数值 | 用途 |
| --- | --- | --- |
| `dur-instant` | 100ms | hover、focus 微反馈 |
| `dur-quick` | 200ms | 状态切换、颜色变化 |
| `dur-normal` | 400ms | 入场、退出 |
| `dur-slow` | 600ms | 翻牌、关键过渡 |
| `dur-ritual` | 1200ms | 仪式性动效（洗牌、Loading）|
| `dur-meditation` | 3000ms | 极慢动效（背景星轨）|

### 8.3 关键动效规范

#### 8.3.1 页面进入
- **容器**：opacity 0→1 + translateY(20px → 0)
- **时长**：400ms
- **缓动**：`ease-mystic`
- **子项 stagger**：80ms

#### 8.3.2 翻牌
- **类型**：Y 轴 3D 旋转
- **角度**：0° → 180°
- **时长**：600ms
- **缓动**：`ease-mystic`
- **细节**：中间点 90° 不可见

#### 8.3.3 洗牌
- **类型**：3D 旋转 + 错位浮动
- **时长**：2500ms
- **缓动**：`ease-meditation`
- **细节**：从慢到快 → 突然停止 → 收拢

#### 8.3.4 抽牌飞出
- **类型**：translateY + 缩放 + 淡出
- **轨迹**：从牌堆到目标位置（贝塞尔曲线）
- **时长**：500ms
- **缓动**：`ease-mystic`

#### 8.3.5 文字打字机
- **速度**：30ms / 字
- **光标**：闪烁 600ms 周期
- **结束**：光标淡出

#### 8.3.6 数字滚动
- **时长**：800ms
- **缓动**：`ease-mystic`
- **用途**：幸运数字显示

### 8.4 动效原则

1. **单一焦点**：同一时刻只有一个主元素在动
2. **不阻塞**：用户始终可操作
3. **可跳过**：仪式性长动效可跳过
4. **降级方案**：尊重 `prefers-reduced-motion`
5. **性能优先**：仅用 `transform` / `opacity`

### 8.5 动效降级

| 原始动效 | 降级方案 |
| --- | --- |
| 翻牌 3D 旋转 | 交叉淡入淡出 |
| 洗牌运动 | 简单淡入 |
| 抽牌飞出 | 淡入到位 |
| 打字机 | 整段淡入 |
| 粒子漂浮 | 静态 |

## 9. Loading（加载态规范）

### 9.1 全屏 Loading

#### 9.1.1 AI 解读加载
- **位置**：屏幕中央
- **视觉**：
  - 中心：旋转的塔罗牌轮廓（SVG，1.2s 周期）
  - 外圈：流动的星光轨道（3 颗星，沿轨道循环）
  - 文字：动态切换的提示文案
- **文字节奏**：

## 10. Mobile Design（移动端规范）

### 10.1 断点

| 断点 | 范围 | 设备 |
| --- | --- | --- |
| `mobile` | < 768px | 手机 |
| `tablet` | 768~1024px | 平板 / 小屏笔电 |
| `desktop` | > 1024px | 桌面 |

### 10.2 移动端布局

- **设计原则**：Mobile First
- **内容宽度**：100% - 24px 边距 × 2
- **垂直节奏**：基于 8px 网格
- **单列布局**：所有页面默认单列

### 10.3 触摸目标

- **最小尺寸**：44 × 44 px（Apple HIG）
- **推荐尺寸**：48 × 48 px
- **间距**：相邻可点击元素至少 8px
- **按钮高度**：最小 48px

### 10.4 移动端导航

- **顶部**：Logo + 返回按钮（条件渲染）
- **底部**：不放导航栏（避免遮挡内容）
- **返回**：依赖浏览器手势 / 顶部返回按钮

### 10.5 移动端字体调整

| 元素 | 桌面 | 移动 |
| --- | --- | --- |
| Hero 标题 | 72px | 48px |
| 页面标题 | 40px | 32px |
| 章节标题 | 28px | 24px |
| 正文 | 15px | 14px（保持阅读）|

### 10.6 移动端间距调整

- 桌面 64px → 移动 24px
- 桌面 96px → 移动 48px

### 10.7 移动端牌尺寸

- 桌面：240 × 360 px
- 移动：160 × 240 px
- 移动端牌间距：8px

### 10.8 移动端输入

- **输入框**：iOS 需 `font-size ≥ 16px` 避免缩放
- **键盘**：自动 focus 触发键盘，页面有"完成"按钮
- **键盘类型**：根据场景使用 `text` / `textarea`

### 10.9 安全区适配

- **iPhone 刘海**：使用 `env(safe-area-inset-top)` / `env(safe-area-inset-bottom)`
- **底部留白**：CTA 按钮距离底部 ≥ 16px + safe-area

## 14. Illustration Style（插画风格）

### 14.1 插画基调

- **风格**：极简几何 + 神秘符号 + 渐变光感
- **不写实**：无具体人物 / 场景
- **抽象**：用形状、光、色块表达概念
- **神秘感**：星座 / 曼陀罗 / 几何 / 光晕

### 14.2 插画元素库

| 元素 | 描述 | 用途 |
| --- | --- | --- |
| 星座图 | 细线连接的星点 | Landing、Loading |
| 曼陀罗 | 同心圆 + 几何图案 | 牌背、Loading |
| 月相 | 8 个月相变化 | 时间、循环 |
| 元素符号 | 火 / 水 / 风 / 土几何化 | 解读页 |
| 星轨 | 弧线 + 流动光 | 背景 |
| 玫瑰 | 几何玫瑰 | 装饰 |
| 蜡烛 | 极简蜡烛 + 火焰 | 仪式性插画 |

### 14.3 插画技术规范

- **格式**：SVG（首选）+ 渐变 + 模糊滤镜
- **颜色**：金色 + 紫色 + 透明
- **不透明度**：整体 0.6~0.9
- **动画**：缓慢旋转 / 脉冲 / 漂浮
- **位置**：背景层、装饰区

### 14.4 插画使用原则

- **克制使用**：每个页面最多 1~2 个插画元素
- **不抢焦点**：插画不与文字竞争视觉
- **不喧宾夺主**：插画色彩不饱和
- **可移除**：即使没有插画，UI 也完整

## 附录 B：设计原则速记卡

```
Mystic    → 暗色 + 金光 + 慢节奏
Elegant   → 留白 + 单一焦点 + 精致细节
Healing   → 圆润 + 暖光 + 不刺眼
Immersive → 全屏 + 仪式动效 + 无干扰
Dark      → 深紫黑为底，金色为光
Golden    → 关键 CTA + 强调 + 装饰
```

## 附录 C：可访问性（a11y）规范

| 维度 | 规范 |
| --- | --- |
| 颜色对比 | 主文字 ≥ 7:1，次文字 ≥ 4.5:1 |
| 焦点环 | 不可隐藏，金色 2px 描边 |
| 触摸目标 | ≥ 44×44 px |
| 屏幕阅读 | 关键元素加 `aria-label` |
| 减少动画 | 尊重 `prefers-reduced-motion` |
| 键盘导航 | Tab 顺序、Enter / Esc 支持 |
| 语义化标签 | 使用 `<button>` `<nav>` `<main>` `<article>` |
| Alt 文本 | 牌面图必须有 alt 描述 |
